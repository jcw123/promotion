# Java 代码规范校验

基于阿里、Google、百度、京东、字节等主流公司的 Java 开发规范，对 Java 源代码进行**全量语法和风格校验**。

## 使用方式

当用户需要对 Java 代码进行规范检查时，调用此技能。支持检查以下内容:

### 检查范围

1. **命名规范**
   - 类名、方法名、变量名、常量名命名风格
   - 包名命名规范

2. **代码格式**
   - 缩进、空格、换行
   - 大括号位置
   - 行长度限制

3. **OOP 规范**
   - 类设计原则
   - 方法设计原则
   - 访问控制修饰符使用

4. **异常处理**
   - 异常捕获规范
   - 异常抛出规范
   - 资源关闭处理

5. **集合与泛型**
   - 集合初始化规范
   - 泛型使用规范

6. **并发编程**
   - 线程安全规范
   - 锁使用规范

7. **注释规范**
   - Javadoc 注释
   - 单行/多行注释

### 输出格式

**全量校验，Top10 展示**: 工具会对代码进行完整的规范检查，发现所有问题，但默认只输出 Top10 最关键的问题（按严重程度排序：错误 > 警告 > 提示）。每个问题包含:
- **问题类型**: 违规类别
- **位置**: 文件 + 行号
- **问题描述**: 具体问题说明
- **原因**: 为什么这是问题
- **修复建议**: 如何修正

### 示例输出

```
=== Java 代码规范校验报告 ===

【Top 1: 命名规范 - 常量命名】
位置：UserService.java:45
问题：常量 "maxRetryCount" 未使用全大写下划线分隔格式
原因：阿里规范强制规定：常量必须使用 UPPER_SNAKE_CASE 格式
修复建议：将 "maxRetryCount" 改为 "MAX_RETRY_COUNT"

【Top 2: 异常处理 - 资源泄漏】
位置：FileReader.java:78
问题：IO 流未使用 try-with-resources 语法
原因：阿里规范推荐：Java7+ 应使用 try-with-resources 自动关闭资源
修复建议：将资源声明放入 try 的括号中，实现自动关闭

...

总计发现 25 个问题，展示 Top 10
```

## 校验规则来源

- 阿里巴巴 Java 开发手册 (强制/推荐/参考)
- Google Java Style Guide
- 百度 Java 开发规范
- 京东 Java 开发规范
- 字节跳动 Java 代码规范

## 执行脚本

运行脚本:
```bash
python3 scripts/java_lint.py --file <java 文件路径> [--output <输出文件>] [--top <输出数量>]
```

参数说明:
- `--file`: 必填，要检查的 Java 文件路径
- `--output`: 可选，结果输出文件路径，默认输出到控制台
- `--top`: 可选，输出的问题数量，默认 10（全量检查，只展示前 N 个）
